# H5 UI Style Guide

更新日期: 2026-03-23

## 目的

基于现有九洲类参考图，固定后续 H5 页面开发的视觉和交互方向，避免前端实现阶段反复改版。

这份文档只定义：

- 视觉风格
- 布局骨架
- 交互层级
- 组件倾向

不定义具体技术实现。

## 总体风格

参考风格的核心特征很明确：

- 浅色底
- 大面积留白
- 黑白灰为主，少量蓝色高亮
- 信息卡片圆角化
- 轻量边框，不依赖厚重阴影
- 页面元素偏“规整面板”，不是强装饰仙侠风
- 重点内容通过字号、选中下划线、标签色来区分

结论：

- 后续 H5 第一版不走重特效、卷轴风、金色浮雕风
- 优先走“现代移动端 + 修仙题材内容”混合路线
- 气质应接近“工具化修仙界面”，不是传统 MMO HUD

## 布局基线

### 1. 顶部资源栏

参考图的稳定结构：

- 左侧头像
- 中间货币/资源胶囊
- 右侧功能图标

建议固定为：

- 左：角色头像
- 中：`灵石`、`铜钱`、后续可扩 `体力`
- 右：`签到`、`邮件`、`设置`、`刷新`

样式要求：

- 资源使用圆角胶囊卡片
- 图标和数字并排
- 图标统一线性风格
- 高度统一，左右间距固定

### 2. 一级页面骨架

推荐统一为：

- 顶部资源栏
- 中部主内容区
- 底部 Tab Bar

建议底部一级导航固定为：

- `战况`
- `地图`
- `秘境`
- `千层塔`
- `更多`

### 3. 页面内二级 Tab

统一规范：

- 文本 tab
- 当前项下方黑色或蓝色短横线
- 不使用厚重背景高亮

适用场景：

- `地图 / 房间`
- `综合 / 世界 / 队伍 / 宗门`
- `信息 / 属性 / 装备 / 功法`
- `基础设置 / 账号安全 / 自动分解 / 兑换`

## 组件风格

### 1. 房间/区域卡片

特征：

- 白底
- 大圆角
- 浅灰边框
- 左标题右推荐境界
- 左下标签，右下补充信息

适用于：

- 区域列表
- 秘境列表
- 活动列表
- 商店/功能建筑卡片

### 2. 信息弹窗

核心操作优先使用中部大弹窗，而不是整页跳转：

- NPC 信息
- 玩家信息
- 千层塔详情
- 签到
- 邮件
- 设置

统一弹窗规范：

- 居中大面板
- 页面背景加半透明遮罩
- 顶部左标题，右关闭
- 内容区可滚动
- 底部操作按钮固定或半固定

### 3. 标签系统

建议统一几类标签：

- 类型标签：`NPC`、`玩家`
- 地域标签：`宁洲`、`东洲`
- 玩法标签：`试炼秘境`、`装备秘境`
- 难度/境界标签：`凡人`、`炼气期`

样式倾向：

- 小尺寸圆角矩形
- 浅蓝底蓝字，或浅灰底黑字

### 4. 按钮系统

固定两套：

- 主按钮：黑底白字，大圆角
- 次按钮：白底黑字，浅灰边框

分别用于：

- 主 CTA：`开始登塔`、`确认修改`
- 次级操作：`交流`、`购买`、`私聊`、`重置`

### 5. 数据卡片

属性页和装备页统一走：

- 双列卡片网格
- 每格一个指标
- 左标签右数值
- 高度统一

## 页面建议

### 1. 地图页

建议直接沿用参考结构：

- 顶部 tab：`地图 / 房间`
- 中部显示当前房间标题、描述、所属地域
- 下部显示房间关系图
- 右下角悬浮圆按钮用于快捷动作

### 2. 世界地图弹窗

统一世界入口建议为：

- `大世界`
- `秘境`
- `活动`

每个 tab 内都是可搜索的卡片列表。

### 3. 房间实体列表

建议结构：

- 顶部显示时间、日期、天气
- 下方为实体卡片列表
- 每项右侧显示类型标签

### 4. “更多”系统页

适合作为系统聚合页，首批入口建议：

- 背包
- 功法
- 境界
- 任务
- 宗门
- 坊市
- 组队
- 排行
- 角色
- 挂机

### 5. 角色 / NPC / 玩家详情

统一模板：

- 顶部头像、类型标签、名称、性别、境界
- 中间 tab
- 下部操作按钮

NPC 建议 tab：

- `信息`
- `描述`

玩家建议 tab：

- `信息`
- `属性`
- `装备`
- `功法`

### 6. 设置页

设置页建议使用弹窗 + tab：

- `基础设置`
- `账号安全`
- `自动分解`
- `兑换`

输入区和开关区都卡片化。

## 动效建议

只保留轻量动效：

- tab 下划线滑动
- modal 淡入 + 轻微上移
- 卡片点击缩放 0.98
- 底部 tab 激活态过渡

不做重粒子或长时特效。

## 实现约束

后续 H5 页面实现时，应尽量遵守：

1. 页面优先移动端竖屏设计。
2. 优先使用白底、浅灰边框、黑字、蓝色高亮。
3. 主交互尽量通过卡片、tab、弹窗完成。
4. 一级导航固定在底部。
5. 详情类内容优先做成统一 modal 模板。
6. 地图、房间、人物、设置、邮箱、签到都保持同一套组件语言。
7. 先追求结构统一，再考虑题材化装饰。

## 对当前项目的直接影响

后续 H5 前端开发时，优先按以下骨架开始：

1. 登录后默认进入 `地图` 页
2. 顶部保留头像、资源、功能图标
3. 底部固定五栏导航
4. `地图` 页先实现：
   - 当前房间卡
   - 房间关系图
   - 房间实体列表
5. `更多` 页先实现：
   - 背包
   - 任务
   - 角色
   - 功法
   - 坊市
6. 详情统一先走弹窗模式

## 后续执行建议

如果开始真正做 H5 页面，建议顺序：

1. 先做低保真线框
2. 再做 design token
3. 再做基础组件：
   - TopBar
   - TabBar
   - MetricPill
   - RoomCard
   - EntityCard
   - ModalPanel
   - ActionButton
4. 最后接地图页和房间页
