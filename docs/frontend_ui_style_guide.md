# H5 UI Style Guide

更新日期: 2026-03-23

## 目的

基于现有九洲类参考图，固定后续 H5 页面开发的视觉和交互方向，避免前端实现阶段反复改版。

这份文档只定义：

- 视觉风格
- 布局骨架
- 交互层级
- 组件倾向

不定义具体技术实现。

## 总体风格

当前参考图已经明确分成两套视觉方向：

- 手机端：浅色、留白、轻卡片
- PC 端：深色、工作台式、三栏布局

统一结论：

- 后续项目不走重特效、卷轴风、金色浮雕风
- 统一走“现代工具化修仙界面”
- 题材表达主要靠文案、插图、标签和结构，不靠厚重仙侠装饰

### 移动端风格

核心特征：

- 浅色底
- 大面积留白
- 黑白灰为主，少量蓝色高亮
- 信息卡片圆角化
- 轻量边框，不依赖厚重阴影
- 重点内容通过字号、选中下划线、标签色来区分

适用范围：

- H5 手机竖屏主客户端
- 轻量弹窗
- 地图 / 房间 / 更多 / 设置等日常页面

### 桌面端风格

核心特征：

- 深色底
- 低饱和黑灰面板
- 蓝色作为激活与焦点色
- 大面板分栏布局
- 背景内容可模糊虚化，强调前景工作台和弹窗
- 信息密度高，但区块边界清楚

适用范围：

- PC 浏览器
- 桌面端测试页
- 后续如果做正式 PC 端网页壳，也默认沿这条线

## 布局基线

### 1. 顶部资源栏

参考图的稳定结构：

- 左侧头像
- 中间货币/资源胶囊
- 右侧功能图标

建议固定为：

- 左：角色头像
- 中：`灵石`、`铜钱`、后续可扩 `体力`
- 右：`签到`、`邮件`、`设置`、`刷新`

样式要求：

- 资源使用圆角胶囊卡片
- 图标和数字并排
- 图标统一线性风格
- 高度统一，左右间距固定

### 2. 一级页面骨架

推荐统一为：

- 顶部资源栏
- 中部主内容区
- 底部 Tab Bar

建议底部一级导航固定为：

- `战况`
- `地图`
- `秘境`
- `千层塔`
- `更多`

### 3. 页面内二级 Tab

统一规范：

- 文本 tab
- 当前项下方黑色或蓝色短横线
- 不使用厚重背景高亮

适用场景：

- `地图 / 房间`
- `综合 / 世界 / 队伍 / 宗门`
- `信息 / 属性 / 装备 / 功法`
- `基础设置 / 账号安全 / 自动分解 / 兑换`

### 4. PC 端主布局骨架

PC 参考图已经很稳定，建议固定为三栏：

- 左栏：角色面板、基础属性、战斗属性、辅助属性
- 中栏：地图 / 房间 / 世界主体内容 + 底部聊天区
- 右栏：系统入口矩阵 + 装备/人物展示 + 快捷动作区

布局要求：

- 左右栏宽度固定或半固定
- 中栏承担主要玩法上下文
- 大屏优先横向展开，不再沿用手机端底部五栏导航
- PC 端保留顶部资源栏，但降低高度

### 5. PC 端弹窗骨架

PC 参考图里的弹窗应作为桌面端标准模板：

- 深色遮罩背景
- 中央大尺寸 modal
- 左侧导航或列表
- 右侧详情
- 顶部标题，右上关闭
- 内容区允许独立滚动

适用场景：

- 大世界 / 秘境 / 活动入口
- 排行榜
- 背包
- 组队
- 设置
- 任务对话
- NPC / 玩家详情

## 组件风格

### 1. 房间/区域卡片

特征：

- 白底
- 大圆角
- 浅灰边框
- 左标题右推荐境界
- 左下标签，右下补充信息

适用于：

- 区域列表
- 秘境列表
- 活动列表
- 商店/功能建筑卡片

### 2. 信息弹窗

核心操作优先使用中部大弹窗，而不是整页跳转：

- NPC 信息
- 玩家信息
- 千层塔详情
- 签到
- 邮件
- 设置

统一弹窗规范：

- 居中大面板
- 页面背景加半透明遮罩
- 顶部左标题，右关闭
- 内容区可滚动
- 底部操作按钮固定或半固定

桌面端补充：

- 优先使用宽弹窗
- 可拆成左右两列
- 左列为分类导航或列表
- 右列为详情区或操作区

### 3. 标签系统

建议统一几类标签：

- 类型标签：`NPC`、`玩家`
- 地域标签：`宁洲`、`东洲`
- 玩法标签：`试炼秘境`、`装备秘境`
- 难度/境界标签：`凡人`、`炼气期`

样式倾向：

- 小尺寸圆角矩形
- 浅蓝底蓝字，或浅灰底黑字

### 4. 按钮系统

固定两套：

- 主按钮：黑底白字，大圆角
- 次按钮：白底黑字，浅灰边框

分别用于：

- 主 CTA：`开始登塔`、`确认修改`
- 次级操作：`交流`、`购买`、`私聊`、`重置`

### 5. 数据卡片

属性页和装备页统一走：

- 双列卡片网格
- 每格一个指标
- 左标签右数值
- 高度统一

PC 端补充：

- 可扩成高密度双列或四列
- 属性列表允许使用表格式块面板
- 装备与背包优先使用网格 + 右侧详情栏

### 6. PC 端系统入口矩阵

PC 参考图里的右栏入口矩阵很适合作为标准系统导航：

- 深色矩形按钮
- 左图标右标题
- 规则网格排列
- 当前不需要彩色大图标

首批保留入口建议：

- 地图
- 秘境
- 千层塔
- 背包
- 功法
- 境界
- 任务
- 坊市
- 排行
- 挂机

### 7. PC 端聊天与战况区

PC 主界面底部应保留聊天 / 战况面板：

- 中栏底部固定聊天区
- 使用 tab 切分 `综合 / 世界 / 队伍 / 宗门 / 私聊 / 战况 / 系统`
- 输入框与发送按钮水平排列
- 聊天区高度可调，但默认不遮挡主地图区

## 页面建议

### 1. 地图页

建议直接沿用参考结构：

- 顶部 tab：`地图 / 房间`
- 中部显示当前房间标题、描述、所属地域
- 下部显示房间关系图
- 右下角悬浮圆按钮用于快捷动作

### 2. 世界地图弹窗

统一世界入口建议为：

- `大世界`
- `秘境`
- `活动`

每个 tab 内都是可搜索的卡片列表。

PC 端补充：

- 左侧列表，右侧详情
- 详情区可显示大图、区域标签、推荐境界、掉落、进入按钮
- 不建议在 PC 端继续沿用手机端单列卡片流

### 3. 房间实体列表

建议结构：

- 顶部显示时间、日期、天气
- 下方为实体卡片列表
- 每项右侧显示类型标签

### 4. “更多”系统页

适合作为系统聚合页，首批入口建议：

- 背包
- 功法
- 境界
- 任务
- 宗门
- 坊市
- 组队
- 排行
- 角色
- 挂机

PC 端补充：

- “更多”不一定作为单独页存在
- PC 端更适合把这些入口直接布在右栏系统矩阵
- 若保留“更多”，应作为系统总览弹窗而不是手机式九宫格页

### 5. 角色 / NPC / 玩家详情

统一模板：

- 顶部头像、类型标签、名称、性别、境界
- 中间 tab
- 下部操作按钮

NPC 建议 tab：

- `信息`
- `描述`

玩家建议 tab：

- `信息`
- `属性`
- `装备`
- `功法`

PC 端补充：

- 玩家详情弹窗优先使用宽面板
- 顶部显示头像、类型、名称、性别、境界
- 中部 tab 切换
- 属性页使用高密度双列指标卡
- 装备页使用双列装备槽
- 功法页使用表格或列表
- 底部操作按钮固定在右下角

### 6. 设置页

设置页建议使用弹窗 + tab：

- `基础设置`
- `账号安全`
- `自动分解`
- `兑换`

输入区和开关区都卡片化。

PC 端补充：

- 设置页优先使用左侧纵向导航
- 右侧内容面板支持长表单与滚动
- 自动分解这类规则页面允许更高信息密度

### 7. 任务对话页

PC 参考图里任务对话已经接近成型，建议固定为：

- 上部对话区
- 右侧或底部选项区
- 下部“下一步 / 返回 / 领取 / 交付”操作区
- 主线、支线状态用标签区分：`进行中 / 已完成 / 未开启`

### 8. 背包页

PC 参考图里的背包结构值得直接沿用：

- 顶部分类 tab
- 次级筛选区：搜索、排序、品质、筛选
- 中部物品网格
- 右侧物品详情
- 底部快捷操作：整理、分解、丢弃、合成

### 9. 排行 / 组队 / 秘境页

PC 端这三类页面适合统一使用“左导航 + 右内容”的后台式结构：

- 左侧竖向分类按钮
- 右侧表格或详情内容
- 视图统一深色表面板
- 主要 CTA 固定在右上或右下

## 动效建议

只保留轻量动效：

- tab 下划线滑动
- modal 淡入 + 轻微上移
- 卡片点击缩放 0.98
- 底部 tab 激活态过渡

不做重粒子或长时特效。

## 实现约束

后续 H5 页面实现时，应尽量遵守：

1. 页面优先移动端竖屏设计。
2. 移动端优先使用白底、浅灰边框、黑字、蓝色高亮。
3. PC 端优先使用深灰背景、低对比面板、蓝色焦点色。
4. 主交互尽量通过卡片、tab、弹窗完成。
5. 手机端一级导航固定在底部。
6. PC 端一级系统入口优先走右栏矩阵或大弹窗。
7. 详情类内容优先做成统一 modal 模板。
8. 地图、房间、人物、设置、邮箱、签到都保持同一套组件语言。
9. 先追求结构统一，再考虑题材化装饰。

## 对当前项目的直接影响

后续 H5 前端开发时，优先按以下骨架开始：

1. 登录后默认进入 `地图` 页
2. 顶部保留头像、资源、功能图标
3. 底部固定五栏导航
4. `地图` 页先实现：
   - 当前房间卡
   - 房间关系图
   - 房间实体列表
5. `更多` 页先实现：
   - 背包
   - 任务
   - 角色
   - 功法
   - 坊市
6. 详情统一先走弹窗模式

后续 PC 页面开发时，优先按以下骨架开始：

1. 左栏角色面板 + 属性区
2. 中栏地图/房间主体 + 底部聊天区
3. 右栏系统入口矩阵 + 人物展示区
4. 世界、秘境、排行、背包、设置统一先走大弹窗
5. NPC / 玩家详情统一用宽 modal
6. 背包、组队、排行、设置尽量复用“左导航 + 右内容”模板

## 后续执行建议

如果开始真正做 H5 页面，建议顺序：

1. 先做低保真线框
2. 再做 design token
3. 再做基础组件：
   - TopBar
   - TabBar
   - MetricPill
   - RoomCard
   - EntityCard
   - ModalPanel
   - ActionButton
4. 最后接地图页和房间页
